## <a id="content1">1、Git介绍</a>

```
版本管理的演变
    ⽤⽬录拷⻉区别不同版本
        公共⽂件容易被覆盖
        成员沟通成本很⾼，代码集成效率低下

    有集中的版本管理服务器，具备⽂件版本管理和分⽀管理能⼒
        集成效率有明显地提⾼
        客户端必须时刻和服务器相连

    分布式 VCS
        服务端和客户端都有完整的版本库
        脱离服务端，客户端照样可以管理版本
        查看历史和版本⽐较等多数操作，都不需要访问服务器
        ⽐集中式 VCS 更能提⾼版本管理效率
```

## <a id="content2">2、Git基础</a>
```
创建Git仓库

```

```
Git仓库状态

    查看状态
        git status 
        git status -s

    仓库状态
        #暂存区
            Changes to be committed:
            (use "git restore --staged <file>..." to unstage)
                new file:   js/script.js
                new file:   styles/style.css


        #工作区
            Changes not staged for commit:
            (use "git add <file>..." to update what will be committed)
            (use "git restore <file>..." to discard changes in working directory)
                modified:   index.html


        #纳入管理，但未被追踪
            Untracked files:
            (use "git add <file>..." to include in what will be committed)
                test.html

    常用指令
        git add .
        git add fileName

        git rm fileName
        git rm --cached fileName
        git rm -r --cached fileName

        git restore fileName
        git restore --staged fileName
```

## <a id="content5">5、Git基础（四）</a>
```
分离头指针
    当我们需要尝试做一些东西的时候，可以使用分离头指针，
    如果最终发现结果不理想可以直接切换到其它分支丢弃提交
    如果最终发现结果是我们想要的可以创建一个新的分支来保存我们的提交
```

```
进一步理解HEAD和branch
    HEAD不仅可以指向某个分支，还可以指向某次具体的commit(分离头指针状态),
    但当HEAD指向某个分支时，也是指向这个分支的最后一次commit
```

## <a id="content7">7、Git独自使用（二）</a>
```
对比文件差异
    工作区与暂存区差异
        git diff
    暂存区与HEAD差异
        git diff --staged

    对比两个分支的不同
        git diff temp master

    对比两次commit的不同
        git diff hash1 hash2

    查看具体文件的不同
        git diff hash1 hash2 -- index.html
```

```
恢复暂存区、工作区
    恢复工作区
        git restore <file>
    恢复暂存区
        git restore --staged <file>
```

```
重置HEAD指向
    git reset HEAD --hard
```

```
删除文件的方法
    git rm fileName
```

## <a id="content8">8、Git独自使用（三）</a>
```
贮藏stash
    贮藏当前内容,并清空暂存区,工作区
        git stash

    查看下stash列表
        git stash list

    应用贮藏,并保留贮藏
        git stash apply

    应用最顶部的贮藏，并清除该贮藏
        git stash pop

```

``` 
忽略.gitignore
    .gitignore文件与.git文件夹平级
    加入.gitignore的文件或文件夹应在未追踪时加入
        否则应该先删除,
        commit,
        重新添加文件,
        加入gitignore文件
```

```
Git的备份
```
